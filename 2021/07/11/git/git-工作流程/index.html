git--初始化本地仓库

git–工作流程

添加文件或提交文件需要有两个基本步骤:
1. git add 将文件添加到staging 暂存区
2. git commit 将暂存区的所有文件推送到仓库

git add

当我们新建 修改并保存一个文件后,本地计算机内的文件便与git仓库中的文件有了不同,因为本地的文件尚未被推送到仓库中

此时git知道我们对文件做出了修改,当使用git status:

可以看出git知道我们新建了文件夹,并建议我们使用git add 将此文件添加到staging工作区

当我们有大量文件需要添加时,使用add .将所有文件添加到暂存区

  • 此处的.代表了所有与仓库不同的文件

git commit

使用git commit来将暂存区的所有文件推送到仓库

  • 在推送时应使用git commit - m"information'',在information中说明此次commit的有关信息,有助于在后续中使用git log查看各次commit的有关信息

直接提交到仓库

在git add -> git commit过程中,git add是将文件放入暂存区中
但当我们知道此文件就是我们需要的最终版本,并可以直接推送到仓库时,add便是一个可以省略的多余步骤

所以我们可以在使用git status检查工作区的修改内容后,直接使用git commit将文件直接推送到仓库中

修改提交

当我们在将文件提交到仓库后又发现了其他微小错误,我们大可以在本地修改源文件后再次进行提交

但是也可以重写最近一次的提交,新的提交会覆盖掉错误的版本

语法为：git commit --amend"commit information

commit后的amend标志告诉git此次提交将会覆盖掉上次提交的内容,上次提交也不会在日志信息汇中出现

永远不要修改非最新提交，因为其他项目团队成员或其他分支有可能是基于该提交的版本。在修改提交之后，他们的版本将会失去了参考，版本控制很难从中恢复。
