git--初始化本地仓库

git–初始化本地仓库

在c盘创建一个Git文件夹,但此文件夹不是必要的
在Git文件夹内创建一个用于学习git的文件夹gitlearn

使用命令git init初始化仓库

随后便可以看见初始化成功的确认信息

git status

在开始向仓库添加文件之前,可以使用``git status`查看当前仓库信息

可见此时仓库是空的,当前branch为master,没有任何commit

尝试向仓库添加一个文本文件text1.txt,并加入一些文本内容

  • 使用 touch filename创建一个文件
  • 使用 ls查看当前路径下的所有文件
  • 使用 vim filename进入vim 编辑器对文件进行编辑
  • 在vim编辑器内按下esc进入,命令模式,键入:wq执行保存后关闭文件退出vim

再次使用git status查看信息,可得:

  • 显示我们新建了一个text1.txt文件,但是该文件未在git的追踪目录内
  • 如果我们不添加和提交此文件,git便不会跟踪此文件

使用git add命令将该文件添加到staging区域,即暂存区

git add text1.txt,可得:

报错提示:The file will have its original line endings in your working directory

  • 是因为git默认将文本中的回车换行统一为CRLF,不影响文本的实际使用

    再次使用git status查看工作区状态

    此时add添加的文件存在暂存区中并未提交

使用git commit将暂存区的文件提交到仓库中

-m后的信息为此次commit的信息

此时再次检查工作区的状态,可以看见干净的工作树

使用git log获得历史提交记录

可见我们设置的提交信息被打印了出来
